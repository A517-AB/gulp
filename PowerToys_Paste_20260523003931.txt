Based on the official package documentation, generated shims, and ecosystem codebase, here is the complete breakdown of the 4 ecosystem packages, the core types, and the API routes used in the Google Labs Jules SDK.

1. The 4 Jules Ecosystem Packages / Folders
The Google Labs Jules ecosystem is structured into four primary modules or packages:

@google/jules (cli): The Command Line Interface wrapper and dashboard tool. It is used to delegate issues directly from the terminal, monitor active tasks, and inspect the remote VMs.
@google/jules-sdk (sdk): The core TypeScript developer client library. It manages network connectivity, session lifecycles, and local SQLite/memory-based caching/querying.
@google/jules-fleet (fleet): A goal-driven continuous automation runner. It parses repository-wide goals (often stored in .fleet/goals/), raises GitHub issues for gaps, and coordinates parallel tasks.
@google/jules-merge (merge): A sequential PR reconciler. It updates branches from the target base branch, verifies CI checks, squashes PRs, and handles automated re-dispatching of sessions when merge conflicts occur.
2. Core API Routes (REST Endpoints)
The ApiClient within the SDK maps its client methods to the following remote REST endpoints:

/sessions
POST /sessions — Creates and starts a new session (interactive or automated).
GET /sessions — Lists historical/active sessions with pagination tokens.
GET /sessions/{id} — Retrieves detailed session status (metadata, active status, duration).
/sessions/{id}:sendMessage
POST /sessions/{id}:sendMessage — Sends a text message or prompt to the coding agent.
/sessions/{id}:approvePlan
POST /sessions/{id}:approvePlan — Approves a pending session plan (advances state from awaitingPlanApproval to inProgress).
/sessions/{id}:archive / /sessions/{id}:unarchive
POST /sessions/{id}:archive — Archives the session to hide it from default listing.
POST /sessions/{id}:unarchive — Restores an archived session.
/sessions/{id}/activities
GET /sessions/{id}/activities — Polls or lists progress events, messages, and artifacts.
GET /sessions/{id}/activities/{activityId} — Retrieves a single, specific activity.
/sources
GET /sources — Lists linked version-control contexts (e.g. GitHub repos).
GET /sources/{name} — Fetches details for a specific repository context.
3. Key TypeScript Types and Interfaces
All of these are exposed by the SDK shim at 
references/jules-sdk.ts
 and defined in 
references/jules-sdk typed.ts
:

A. Client Orchestration
JulesClient: Connects to the service, holds configurations, and exposes high-level session factories (session, run, all, select).
SessionClient: Scoped wrapper for a single session ID; provides control commands (approve(), send(), ask(), waitFor(), stream()).
ActivityClient: Handles polling and streaming of history or live updates (history(), updates(), stream()).
B. Sessions & Outcomes
SessionConfig: Initialization payload containing prompt, source inputs, and flags like autoPr and requirePlanApproval.
SessionResource: Hydrated session state metadata including creation timestamp, repository identifiers, and current state.
SessionState: Union representing state transitions: 'awaitingPlanApproval' | 'inProgress' | 'completed' | 'failed' | 'cancelled'.
SessionOutcome / Outcome: The final result containing target branch configurations and the generated pullRequest details.
C. Activities (Session Timeline Events)
Activity: Base interface containing originator, timestamp, type, and associated artifacts.
ActivityUserMessaged / ActivityAgentMessaged: Exchanged messages between the developer and the agent.
ActivityPlanGenerated / ActivityPlanApproved: Details of the agent's proposed plan step-list.
ActivityProgressUpdated: Action progress logs showing what tool the agent is executing.
ActivitySessionCompleted / ActivitySessionFailed: Emitted when the session reaches a terminal state.
D. Artifacts (Outputs)
Artifact: Base interface representing files, terminal outputs, or media generated by the agent.
ChangeSetArtifact: Represents diff code modifications. Provides parsed() to extract additions/deletions and change types per file path.
BashArtifact: Contains outputs from executed terminal operations (stdout, stderr, and exitCode).
MediaArtifact: Base64 encoded binary content (e.g. browser screenshots) with a save(filepath) method.
