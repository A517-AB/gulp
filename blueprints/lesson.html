TypeScript Utility Types

How to customize imported types without rewriting them.

The Problem

You imported the Cheese type from someone else's package. It looks like this:

import type { Cheese } from 'some-package';

// What it looks like under the hood:
// type Cheese = {
//   name: string;
//   ageInMonths: number;
//   isStinky: boolean;
// };

But you don't care if it's stinky! You just want to use the package as-is, immediately, without having to check the isStinky property or redefine the whole type from scratch.

The Solution: Omit

TypeScript has built-in "Utility Types" that let you transform existing types. To remove a property, you use Omit.

Omit takes two arguments: the original type, and the property you want to throw away.

// Create a new type based on Cheese, but remove 'isStinky'
type MyPerfectCheese = Omit<Cheese, 'isStinky'>;

// Now, MyPerfectCheese looks exactly like this:
// type MyPerfectCheese = {
//   name: string;
//   ageInMonths: number;
// }

In Action

Now you can consume it straight from the package without TypeScript yelling at you about missing the stinky property!

const snackTime: MyPerfectCheese = {
  name: "Gouda",
  ageInMonths: 6
  // No 'isStinky' required here!
};

console.log(`Eating ${snackTime.name} immediately!`);
